# Chugalkhor Bandar Bootstrap Canon

Welcome to the canonical bootstrap repository for the **Chugalkhor Bandar** universe.

This repository is the authoritative source of truth for the initial state of the Jungle world.

---

# Purpose

The bootstrap exists to:

- Define the world.
- Record stable canon.
- Preserve important history.
- Maintain consistency between stories.
- Serve as the foundation for future applications and storytelling.

Every new story should build upon this repository instead of replacing it.

# Canon

Everything inside the bootstrap folder is considered canonical unless explicitly revised.

If two files appear to disagree:

1. `world-rules.md`
2. `canon.md`

take precedence.

---

# Adding New Stories

When introducing a new story:

1. Create a new file in `/stories/`.
2. Record permanent consequences.
3. Update `world-timeline.md`.
4. Update affected characters.
5. Update relationships if necessary.
6. Update territories or places if ownership changes.

---

# Revising Canon

Canon may evolve.

Do **not** silently overwrite history.

Instead:

- revise affected files,
- preserve chronology,
- document permanent changes.

---

# Unknown Information

Unknown facts should remain unknown.

Never invent missing history unless explicitly approved.

Use:

```
TODO
```

or

```
Unknown
```

instead of guessing.

---

# Stable IDs

Every canonical entity should have a stable identifier.

Examples:

```
character_hippu_king
place_home_jungle
territory_hippu_kingdom
story_founding_hippu_dynasty
organization_hippu_government
object_hidden_treasure
resource_hippu_coins
relationship_001
```

Stable IDs should never change once published.

---

# Naming

Prefer descriptive names over abbreviations.

Examples:

Good:

- Hippu Palace
- Border Jungle
- Ancient Rabbitu Jungle

Avoid:

- HP
- BJ
- ARJ

---

# Stories

Stories are historical events.

They should answer:

- What happened?
- Who participated?
- What permanently changed?

Every story should leave the world in a well-defined new state.

---

# Characters

Character files describe long-term identity.

Temporary events belong in stories unless they permanently change the character.

---

# Places

Places describe locations.

Political ownership belongs in territories.

Organizations belong in organizations.

---

# Relationships

Relationships describe long-term connections.

Temporary alliances or conflicts should be documented in stories unless they become permanent.

---

# Philosophy

This world is designed to feel:

- warm,
- humorous,
- adventurous,
- historical,
- internally consistent.

Comedy is encouraged.

Cruelty should be rare.

Character growth is preferred over spectacle.

---

# Future Expansion

Expected future additions include:

- New kingdoms
- New species
- New stories
- Festivals
- Laws
- Customs
- Historical discoveries
- Objects
- Family members

The world should expand without breaking existing canon.
